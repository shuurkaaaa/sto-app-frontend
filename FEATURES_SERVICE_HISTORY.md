# Нові Функції: Історія Обслуговування та Нагадування про ТО

## 📋 Огляд змін

Реалізовано комплексну систему управління історією обслуговування клієнтів з автоматичними нагадуваннями про технічне обслуговування.

## ✨ Реалізовані Функціональні Можливості

### 1. **Кнопка "Історія Обслуговування" на Картці Клієнта**

**Розташування:** Внизу кожної картки клієнта на сторінці `/customers`

**Функція:** При натисканні відкривається модальне вікно з повною історією всіх ремонтів та візитів клієнта до СТО

**Кнопка:**
```jsx
📋 Історія обслуговування
```

---

### 2. **Модальне Вікно ServiceHistoryModal**

**Файл:** `/src/Components/CustomerComponents/ServiceHistoryModal.js`

**Функціональність:**

- **Сортування:** Всі замовлення сортуються від найновіших до найстаріших
- **Деталі кожного замовлення включають:**
  - Статус (Завершено, В роботі, Готово, Очікування, Скасовано)
  - Дата і час обслуговування
  - Кількість днів/годин тому обслуговування було виконано
  - Вартість замовлення
  - Деталі автомобіля (марка, модель, номер)
  - Перелік всіх послуг з їх цінами
  - Примітки спеціаліста
  - Спосіб оплати
  - Позначка "⚡ Екстрене" якщо замовлення було термінове

**Приклад відображення статусу:**
- 🟢 **Завершено** - зелений колір
- 🔵 **В роботі** - синій колір  
- 🟠 **Готово** - помаранчевий колір
- 🟤 **Очікування** - сірий колір
- 🔴 **Скасовано** - червоний колір

---

### 3. **Система Нагадування про ТО (6-місячний цикл)**

**Логіка:** При завантаженні сторінки Customers або при оновленні списку клієнтів система автоматично перевіряє:

- Дату останнього візиту кожного клієнта
- Якщо з останнього завершеного замовлення пройшло **більше 180 днів (6 місяців)** → клієнту потрібне ТО

**Функція перевірки:** `checkCustomerServiceReminder()` в `/src/utils/customerReminder.js`

```javascript
// Повертає об'єкт з інформацією:
{
  needsReminder: true/false,
  daysSinceService: кількість днів,
  monthsSinceService: кількість місяців,
  lastServiceDate: дата останнього обслуговування,
  message: текстове повідомлення
}
```

---

### 4. **Червона Позначка на Картці Клієнта**

**Розташування:** Верхній-правий кут картки

**Умова Відображення:** Якщо `daysSinceService > 180` днів

**Вигляд:** 
```
┌─────────────────────┐
│ ТО ПОТРІБНЕ         │  ← червона позначка
│ Останнє обслуговування...
│
```

**Важливо:** Позначка відображається **незалежно від того, чи картка клієнта в архіві чи ні**

---

### 5. **Сповіщення про ТО в Панелі Notifications**

**Файл:** `/src/Context/NotificationsContext.js`

**Нова функція:** `notifyMaintenanceNeeded(customer)`

**Як працює:**

1. При завантаженні сторінки Customers перевіряються всі клієнти
2. Для клієнтів з `needsReminder: true` генерується сповіщення
3. Сповіщення відображається як toast-повідомлення (верхній-правий кут)
4. Сповіщення також добавляється в список всіх сповіщень на сторінці `/notifications`

**Toast-повідомлення:**
```
⚡ ОБСЛУГОВУВАННЯ ПОТРІБНЕ
Ім'я Клієнта
Час запропонувати клієнту технічне обслуговування
```

**Тип сповіщення:** `maintenance` з пріоритетом `high` (червоний колір)

### 6. **Додано Поле VIN-Коду при Реєстрації**

**Де додано:**

#### A. **AddCustomerModal** (первинна реєстрація клієнта)
- **Файл:** `/src/Components/CustomerComponents/AddCustomerModal.js`
- **Поле:** "VIN-код (необов'язково)"
- **Тип:** Текстове поле (необов'язкове заповнення)
- **Значення за замовчуванням:** якщо не заповнено → `"—"`

#### B. **CustomerDetailsModal** (додавання другої машини)
- **Файл:** `/src/Components/CustomerComponents/CustomerDetailsModal.js`
- **Поле:** "VIN-код (необов'язково)"
- **Тип:** Текстове поле (необов'язкове заповнення)
- **Розташування:** При натисканні "+ Додати автомобіль" форма розширюється і містить VIN-поле

**Форма додавання машини:**
```
┌─────────────────────────────────┐
│ Марка:        [Введіть марку]  │
│ Модель:       [Введіть модель] │
│ Держ. номер:  [AA 1234 AA]    │
│ VIN-код:      [Необов'язково]  │
│ [Зберегти]  [Скасувати]        │
└─────────────────────────────────┘
```

4. **День 15.05.2025:** Іван прийшов на ТО → утворено нове PENDING замовлення
