### 📋 Структура даних

#### VIN-код у замовленні:
```json
{
  "id": 1,
  "customerId": 5,
  "carDetails": "Toyota Camry (АА1234ББ)",
  "vinCode": "JT2BF22K3M0000001",
  "totalPrice": 1500,
  "createdAt": "2026-05-10T14:30:00Z"
}
```

#### VIN у автомобілі:
```json
{
  "id": 12,
  "brand": "Toyota",
  "model": "Camry",
  "plate": "АА1234ББ",
  "vin": "JT2BF22K3M0000001",
  "customerId": 5
}
```

#### Відповідь перевірки VIN:
```json
{
  "success": true,
  "validation": {
    "valid": true,
    "message": "VIN-код валідний"
  },
  "decode": {
    "success": true,
    "source": "NHTSA",
    "vin": "JT2BF22K3M0000001",
    "make": "Toyota",
    "model": "Camry",
    "year": "2000"
  },
  "history": [
    {
      "date": "2026-02-10T14:30:00Z",
      "service": "ТО-2",
      "mileage": 45000,
      "notes": "Замінено масло, фільтри, свічки"
    }
  ]
}
```

### 🔗 API Endpoints

| Метод | Endpoint | Опис |
|-------|----------|------|
| `POST` | `/api/vin/decode` | Розшифрування VIN-коду |
| `POST` | `/api/vin/validate` | Валідація VIN |
| `GET` | `/api/vin/history/:vin` | Історія обслуговування |
| `POST` | `/api/vin/check` | Повна перевірка (валідація + decode + історія) |

**Автентифікація:** Всі запити вимагають JWT токена в заголовку `Authorization: Bearer <token>`

### 🧪 Тестування

#### Тестовий VIN для розшифрування:
```
JT2BF22K3M0000001  ← Toyota Camry 2000
LFVNCLF30LP078915  ← Lexus
```

#### Кроки для перевірки:
1. Відкрити "Dashboard" → "Нове замовлення"
2. Ввести VIN-код в поле (опціонально)
3. Створити замовлення
4. Перейти на сторінку "Customers"
5. Натиснути на VIN-код на картці клієнта
6. Переглянути інформацію та історію
