## 📋 Що Повинно Працювати

| **VIN при додаванні машини** | + Додати автомобіль | Поле VIN видима |

## 🎯 Основні Кейси

### Кейс 1: Історія Обслуговування
```
Customers
  ↓
Картка клієнта
  ↓
Натиснути: 📋 Історія обслуговування
  ↓
ServiceHistoryModal відкривається
  ↓
✅ Бачимо всі замовлення з деталями
```

### Кейс 2: Червоне ТО Нагадування
```
Клієнт якого останній візит > 180 днів
  ↓
На його картці: червона позначка "ТО ПОТРІБНЕ"
  ↓
При завантаженні сторінки: Toast-сповіщення
  ↓
В "Сповіщення": запис про ТО
```

### Кейс 3: Додавання VIN
```
Нова реєстрація клієнта
  ↓
Заповнити форму
  ↓
Поле "VIN-код" (необов'язкове)
  ↓
Натиснути "Зареєструвати"
  ↓
✅ Клієнт створено з VIN
```

## 🐛 Якщо Щось Не Працює

### Проблема: Позначка не видима
```
✓ Перевірити: customer.orders?.length > 0
✓ Перевірити: daysSinceService > 180
✓ Відкрити DevTools (F12) - Console
✓ Перезавантажити сторінку
```

### Проблема: Кнопка "Історія" не видима
```
✓ Убедитесь что картка рендерится
✓ Прокрутите вниз - кнопка внизу
✓ Перезавантажите F5
✓ Очистить кеш: Ctrl+Shift+Delete
```

### Проблема: Сповіщення не приходить
```
✓ Перевірити: checkCustomerServiceReminder() повертає true
✓ Перевірити: Backend запущений
✓ Перевірити: useNotifications hook підключений
```

### Проблема: VIN не зберігається
```
✓ Перевірити: Browser DevTools → Network
✓ Перевірити: POST запит містить vin
✓ Перевірити: Backend /api/cars прийняв VIN
✓ Перевірити: SQLite Schema Car.vin існує
```
