# 🧪 Інструкція Тестування Нових Функцій

## Передумови

- React сервер запущений на `http://localhost:3000`
- Backend запущений на `http://localhost:5000`
- Ви залогінені в системі

## 1️⃣ Тестування Кнопки "Історія Обслуговування"

### Крок 1: Перейти на сторінку Customers
```
1. Натиснути "Клієнти" в навбарі
2. Переконатися що список клієнтів завантажився
```

### Крок 2: Знайти кнопку на картці
```
1. Прокрутити вниз картки клієнтів
2. Внизу кожної картки повинна бути кнопка: "📋 Історія обслуговування"
3. Натиснути на цю кнопку
```

### Крок 3: Перевірити модальне вікно
```
✅ Повинне відкритися модальне вікно з заголовком "Історія обслуговування"
✅ Содержер список всіх замовлень клієнта (від новіших до старіших)
✅ Кожне замовлення показує:
   - Статус з відповідним кольором
   - Дату та час
   - "X днів тому"
   - Вартість
   - Автомобіль (марка, модель, номер)
   - Список послуг
   - Примітки
   - Спосіб оплати
   - ⚡ Екстрене (якщо було)
```

### Крок 4: Закрити модальне вікно
```
1. Натиснути кнопку "Закрити" 
   або
2. Натиснути крестик (✕) у верхньому правому кутку
   або
3. Натиснути за межами вікна
```

## 2️⃣ Тестування Червоної Позначки "ТО ПОТРІБНЕ"

### Передумови
Потрібен клієнт у якого останній візит був **більше 180 днів тому** (більше 6 місяців)

> ℹ️ Якщо у вас нема такого клієнта, ви можете:
> - Вручну змінити дату в БД, або
> - Додати дані через API, або
> - Читайте розділ "Як створити тестові дані" нижче

### Крок 1: Переглянути картку
```
1. На сторінці Customers прокрутіть до картки клієнта
2. Якщо останній візит > 6 місяців тому, повинна бути
   позначка в верхньому правому куті картки:
   
   ╔═══════════════════╗
   ║  ТО ПОТРІБНЕ ▲    ║  ← красна позначка
   ║                   ║
   ║ Ім'я Клієнта      ║
   ╚═══════════════════╝
```

### Крок 2: Перевірити стиль
```
✅ Позначка має червоний фон (#ef4444)
✅ Текст білий
✅ Шрифт маленький та жирний
✅ Розташована у верхньому правому куту картки
```

### Крок 3: Перевірити на картці
```
✅ Позначка видима на АКТИВНИХ картках
✅ Позначка видима на АРХІВНИХ картках (якщо > 6 місяців)
✅ Позначка видима НЕЗАЛЕЖНО від VIP статусу
```

### Крок 4: Перевірити текст із підсказкою
```
1. Найміте на картку (натисніть на неї)
2. Відкриється CustomerDetailsModal
3. Внизу картки повинен бути текст типу:
   "Останнє обслуговування 8 місяців тому (01.11.2024). 
    Час на технічне обслуговування!"
```

## 3️⃣ Тестування Сповіщень про ТО

### Крок 1: Завантажити сторінку Customers
```
1. Натиснути "Клієнти" в навбарі (або оновити сторінку F5)
2. Система автоматично перевіртає всіх клієнтів
3. Якщо є клієнти з ТО потрібним > 6 місяців, з'являються toast-сповіщення
```

### Крок 2: Перевірити toast-повідомлення
```
✅ Сповіщення з'являються у верхньому правому куті
✅ Формат:
   ╔════════════════════════════════╗
   ║ ⚡ ОБСЛУГОВУВАННЯ ПОТРІБНЕ     ║
   ║ Ім'я Клієнта                   ║
   ║ Час запропонувати клієнту ТО   ║
   └════════════════════════════════┘
   
✅ Сповіщення зникає через 3 секунди
✅ Червоний колір бордюру
```

### Крок 3: Перевірити у вкладці Notifications
```
1. Натиснути "Сповіщення" в навбарі
2. Повинен бути список сповіщень
3. Сповіщення про ТО повинно бути там з типом "maintenance"
4. Пріоритет: HIGH (червоний)
5. Можна видалити сповіщення натисканням на нього
```

## 4️⃣ Тестування VIN-Поля при Реєстрації

### Тест 1: Додавання нового клієнта з VIN

#### Крок 1: Натиснути "Нова реєстрація"
```
1. На сторінці Customers натиснути кнопку "+ Нова реєстрація"
2. Відкриється AddCustomerModal
```

#### Крок 2: Заповнити форму
```
- ПІБ: Петро Коваль
- Телефон: +380971234567
- Марка: Toyota
- Модель: Camry
- Держ. номер: АВ 1234 СД
- VIN-код: JTDKARFP4L2050166  ← новое поле!
- Звідки дізналися: Прямий візит
```

#### Крок 3: Натиснути "Зареєструвати"
```
✅ Клієнт додатися
✅ Modal закривается
✅ Новий клієнт з'являється у списку
```

#### Крок 4: Перевірити збереження VIN
```
1. Натиснути на картку нового клієнта
2. Відкриється CustomerDetailsModal
3. Повинна бути машина з VIN-кодом
4. При наведенні на VIN-кнопку показується повний код
```

### Тест 2: Додавання другої машини з VIN

#### Крок 1: Відкрити деталі клієнта
```
1. На сторінці Customers натиснути на картку клієнта
2. Відкриється CustomerDetailsModal
```

#### Крок 2: Натиснути "+ Додати автомобіль"
```
1. На лівій стороні модалу натиснути "+ Додати автомобіль"
2. Форма розширяется і показує поля для введення
```

#### Крок 3: Заповнити форму
```
- Марка: BMW
- Модель: X5
- Держ. номер: БВ 5678 ЕФ
- VIN-код: WBAEV91090EX12345  ← необов'язково!
```

#### Крок 4: Натиснути "Зберегти"
```
✅ Машина додатися до списку
✅ Форма закривается
✅ Новая машина видима в списку
```

#### Крок 5: Тестування без VIN
```
1. Натиснути "+ Додати автомобіль" знову
2. Заповнити ВСЕ крім VIN:
   - Марка: Mercedes
   - Модель: E-Class
   - Держ. номер: ГХ 9999 ІЙ
   - VIN-код: [ПОРОЖНЬО]

4. Машина повинна додатися і без VIN (значення: "—")
```

## 5️⃣ Як Створити Тестові Дані

### Сценарій: Клієнт з давньої сервісної історією

#### Опція 1: Вручну через UI (повільно)
```javascript
// Тестові дані для завантаження
const testCustomer = {
  name: "Тестовий Клієнт",
  phone: "+380991234567",
  cars: [{
    brand: "BMW",
    model: "X5",
    plate: "AA 1234 BB",
    vin: "WBAEV91090EX00000"
  }],
  orders: [
    {
      status: "COMPLETED",
      createdAt: new Date(Date.now() - 200*24*60*60*1000), // 200 днів тому
      services: [{name: "ТО-1", price: 1000}],
      totalPrice: 1000
    }
  ]
}
```

#### Опція 2: Через API (швидше)
```bash
curl -X POST http://localhost:5000/api/customers \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -d '{
    "name": "Тестовий ТО",
    "phone": "+380998765432",
    "source": "Тест"
  }'

# Тоді додати машину:
curl -X POST http://localhost:5000/api/cars \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -d '{
    "customerId": CUSTOMER_ID,
    "brand": "Toyota",
    "model": "Camry",
    "plate": "AB 5678 CD",
    "vin": "JTDKARFP4L2050166"
  }'
```

## 🔍 Налагодження (Debugging)

### Проблема: Позначка "ТО ПОТРІБНЕ" не видима

**Рішення:**
```javascript
// Перевірте в консолі браузера:
// Клієнт повинен мати orders з COMPLETED статусом
// Останній order.completedAt < Date.now() - 180*24*60*60*1000

// Коректна логіка:
daysSinceService > 180 → needsReminder = true
daysSinceService ≤ 180 → needsReminder = false
```

### Проблема: Сповіщення не приходить

**Рішення:**
```javascript
// Перевірте:
1. useNotifications хук правильно підключений
2. checkCustomerServiceReminder повертає needsReminder: true
3. notifyMaintenanceNeeded(customer) викликається
4. Toast-повідомлення мають ID на основі customer.id
```

### Проблема: VIN-поле не зберігається

**Рішення:**
```javascript
// Перевірте у консолі:
1. FormData містить vin: "..."
2. API має прийняти car.vin
3. Backend schema Car.vin існує
4. Не використовується стара логіка: vin: "—"
```

## 📋 Чек-лист Тестування

- [ ] Кнопка "Історія обслуговування" видима на картці
- [ ] Модальне вікно відкривається при натисканні
- [ ] Модальне вікно показує всі замовлення клієнта
- [ ] Замовлення сортуються від новіших до старіших
- [ ] Кожне замовлення показує правильний статус з кольором
- [ ] Кожне замовлення показує послуги, примітки, спосіб оплати
- [ ] Позначка "ТО ПОТРІБНЕ" видима при > 6 місяців
- [ ] Позначка червона і розташована вгорі справа
- [ ] Сповіщення приходить при завантаженні сторінки
- [ ] Сповіщення видима в Notifications вкладці
- [ ] VIN-поле видима при реєстрації клієнта
- [ ] VIN-поле видима при додаванні другої машини
- [ ] VIN-поле необов'язкове для заповнення
- [ ] VIN зберігається і відображається правильно
- [ ] Modal закривается натисканням на крестик
- [ ] Modal закривается натисканням на "Закрити"
- [ ] Modal закривается натисканням за межами вікна
- [ ] Все функціонує на активних і архівних картках
